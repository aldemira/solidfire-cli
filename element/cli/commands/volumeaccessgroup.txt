         <title>
            sfcli VolumeAccessGroup modifylunassignments
            <varname cid="1IOAZZ">options</varname>
         </title>
         <table>
            <tgroup cols="2" cid="Nirny">
               <colspec colnum="1" colname="col1" colwidth="1.34*" cid="21O9D4" />
               <colspec colnum="2" colname="col2" colwidth="1.00*" cid="EKRMv" />
               <thead cid="23Fc7a">
                  <row cid="13uyDN">
                     <entry colname="col1" cid="2x5tN">Description</entry>
                     <entry colname="col2" cid="13UHCG">Options</entry>
                  </row>
               </thead>
               <tbody cid="7uqUX">
                  <row cid="2EBjiH">
                     <entry colname="col1" cid="1Thqa5">
                        The <apiname>ModifyVolumeAccessGroupLunAssignments</apiname> method enables you to define custom LUN assignments for specific volumes. This method changes only LUN values set on the <parmname>lunAssignments</parmname parameter in the volume access group. All other LUN assignments remain unchanged. LUN assignment values must be unique for volumes in a volume access group. You cannot define duplicate LUN values within a volume access group. However, you can use the same LUN values again in different volume access groups.  <note>: Correct LUN values are 0 through 16383. The system generates an exception if you pass a LUN value outside of this range. None of the specified LUN assignments are modified if there is an exception.  Caution: If you change a LUN assignment for a volume with active I/O, the I/O can be disrupted. You might need to change the server configuration before changing volume LUN assignments. 
                     </entry>
                     <entry colname="col2" cid="sKwWv">
                        <parmname>volumeaccessgroupid</parmname>
                        : The ID of the volume access group for which the LUN assignments will be modified. 
                        <parmname>lunassignments</parmname>
                        : The volume IDs with new assigned LUN values. 
                     </entry>
                  </row>
               </tbody>
            </tgroup>
         </table>

                        : The ID of the SolidFire virtual network to associate the volume access group with. 
                        <parmname>name</parmname>
                        : The new <parmname>name</parmname for this volume access group. Not required to be unique, but recommended. 
                        <parmname>initiators</parmname>
                        : List of <parmname>initiators</parmname to include in the volume access group. If unspecified, the access group's configured <parmname>initiators</parmname are not modified. 
                        <parmname>volumes</parmname>
                        : List of <parmname>volumes</parmname to initially include in the volume access group. If unspecified, the access group's <parmname>volumes</parmname are not modified. 
                        <parmname>deleteorphaninitiators</parmname>
                        : true: Delete initiator objects after they are removed from a volume access group. false: Do not delete initiator objects after they are removed from a volume access group. 
                        <parmname>attributes</parmname>
                        : List of <parmname>name</parmname-value pairs in JSON object format.
