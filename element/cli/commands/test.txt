         <title>
            sfcli Test ping
            <varname cid="1IOAZZ">options</varname>
         </title>
         <table>
            <tgroup cols="2" cid="Nirny">
               <colspec colnum="1" colname="col1" colwidth="1.34*" cid="21O9D4" />
               <colspec colnum="2" colname="col2" colwidth="1.00*" cid="EKRMv" />
               <thead cid="23Fc7a">
                  <row cid="13uyDN">
                     <entry colname="col1" cid="2x5tN">Description</entry>
                     <entry colname="col2" cid="13UHCG">Options</entry>
                  </row>
               </thead>
               <tbody cid="7uqUX">
                  <row cid="2EBjiH">
                     <entry colname="col1" cid="1Thqa5">
                        You can use the <apiname>TestPing</apiname> API method to validate the connection to all the nodes in a cluster on both 1G and 10G interfaces by using ICMP packets. The test uses the appropriate MTU sizes for each packet based on the MTU settings in the network configuration. <note>: This method is available only through the per-node API endpoint 5.0 or later. 
                     </entry>
                     <entry colname="col2" cid="sKwWv">
                        <parmname>attempts</parmname>
                        : Specifies the number of times the system should repeat the test ping. The default value is 5. 
                        <parmname>hosts</parmname>
                        : Specifies a comma-separated list of addresses or hostnames of devices to ping. 
                        <parmname>totaltimeoutsec</parmname>
                        : Specifies the length of time the ping should wait for a system response before issuing the next ping attempt or ending the process. 
                        <parmname>packetsize</parmname>
                        : Specifies the number of bytes to send in the ICMP packet that is sent to each IP. The number must be less than the maximum MTU specified in the network configuration. 
                        <parmname>pingtimeoutmsec</parmname>
                        : Specifies the number of milliseconds to wait for each individual ping response. The default value is 500 ms. 
                        <parmname>prohibitfragmentation</parmname>
                        : Specifies that the Do not Fragment (DF) flag is enabled for the ICMP packets. 
                     </entry>
                  </row>
               </tbody>
            </tgroup>
         </table>
      </section>
